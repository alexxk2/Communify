# Communify

Приложение - список контактов с которыми можно связаться. Приложение состоит из 5 экранов.
Реализована регистрация и вход пользователя с сохранением данных для следующего входа (данные хранятся в базе данных). Основные экраны - вход/регистрация, контакты, детали контакта, профиль.<br>
<br>**Стартовый экран и  экраны регистрации/входа**
<br>В приложении реализован стартовый экран на котором пользователь может перейти к регистрации/входу, либо войти в приложение как гость(будет ограничен функционал). Экран регистрации валидирует вводимые значения и сохраняет 
данные пользователя в базу данных, если пользователь с таким же логином уже есть в базе, то будет показано сообщение об этом. На экране входа нужно ввести логин/пароль и приложение проверит - есть ли в базе данных такой пользователь и правильный ли пароль.<br>
<br>**Экран "Контакты"**
<br> На экране отображаются список сгенерированных контактов.  На экране также присутствуют теги (chip group) с помощью которых можно отфильтровать
 список отображемых контактов. На каждом элементе есть кнопки отправить сообщение и позвонить. При выходе из приложения выбранные контакты сохраняются в базе данных, также по свайпу можно обновить контакты.<br>
<br>**Экран "Детали"**
<br>При клике на контакт пользователь попадет на экран деталей, где сможет посмотреть подробности контакта, а также открыть карту и посмотреть в каком городе живет человек, отправить сообщение, позвонить или скопировать нужные данные.<br>
<br>**Экран "Профиль"**
<br>На экране профиля пользователь видит данные о себе, аватар профиля, имя, логин и пароль. Аватар, логин и пароль можно обновить(поменять). Обновление аватара в базе данных происходит автоматически при выборе нового, 
изменение логина/пароля надо подтверждать. Пользователь может удалить свой профиль, в этом случае стираются все данные о профиле из базы данных, а также все сохраненные контакты.<br>

## Технические детали реализации
+ Приложение создано по принципам чистой архитектуры и SOLID, есть 3 слоя (data, domain, presentation). Связь между слоями осуществляется через интерфейсы.
+ Используется паттерн MVVM в презентейшн слое.
+ В качетсве библиотеки для DI выбрана библиотека Hilt.
+ Приложения реализовано по принципу единой активити с фрагментами, навигация через navigation component и safe args.
+ Получение данных реализовано с помощью Retrofit и корутин.
+ Контакты и профиль пользователя хранятся в SQlite базе данных, для работы с ней используется библиотека Room и корутины.
+ Списки сделаны через recyclerView и ListAdapter + Diffutil.
+ Все по работе с данными лежит в data слое, некоторая часть бизнес логики (фильтрация) в domain.
+ На экране каталога и избранного реализована LCE модель управления состоянием экрана (loading, content, error).
